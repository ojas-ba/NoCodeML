# Screenshots

## Adding Your Screenshots

Place your project screenshots in this folder to display them in the main README.

### Required Screenshots

1. **landing-page.png** - Main landing page of the application
   - Recommended size: 1920x1080 or 1280x720
   - Format: PNG or JPG
   - Shows: Login/Register page or home dashboard

### Optional Screenshots (you can add more)

2. **dashboard.png** - Main dashboard view
3. **dataset-upload.png** - Dataset upload interface
4. **eda-analysis.png** - EDA visualization page
5. **model-training.png** - Model training interface
6. **results.png** - Training results and metrics
7. **predictions.png** - Prediction interface

### Tips for Good Screenshots

- Use high resolution (minimum 1280x720)
- Capture full browser window
- Show actual data/visualizations
- Ensure UI is clean (no debug tools visible)
- Use light mode for better visibility
- Compress images to keep repository size reasonable

### How to Reference in README

```markdown
![Screenshot Description](./screenshots/filename.png)
*Caption for the screenshot*
```

### Image Optimization

Before adding screenshots, consider optimizing them:
- Use PNG for UI screenshots (better quality)
- Use JPG for photos/complex images (smaller size)
- Compress with tools like TinyPNG or ImageOptim
- Keep individual images under 500KB when possible
